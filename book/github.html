github上传文件

git init 初始化一个git仓库 git status 查看文件状态 git add . 把编辑区放在暂存区 git commit -m “ ” 将暂存区文件提交到分支区 git remote + 自己项目的地址（本行复制就可以 git push origin master 同步文件 git pull github上的版本与本地版本不一样是需要先pull下来更新本地版本 git clone+链接（从github clone or download 复制） 从github下载本地没有的文件 git log 查询commit git reflog 查询近期操作 git reset --hard 版本号 版本控制

分支

当主分支上的内容需要修改但不能直接修改改变上线作品，可以建一个分支，在分支上修改然后合并两个分支内容，就不会影响页面内容 master是一个分支 git branch+分支名 建一个分支 git branch 查看分支 git checkout+分支名 切换分支 合并分支需要先切换到主分支然后用： git merge 分支名 主分支和次分支同时添加内容合并就会产生冲突，需要先切换到主分支，分支里的内容需要先提交然后合并 git branch -d 分支名 删除分支————只是删除本地分支 git push origin ：分支名 删除远程分支 *github里也可操作，但涉及版本问题需先pull，建议都本地操作
